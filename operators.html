Performing Mathematical and Logical Operations on Data

Numeric Operators

There are many operaitons that can be performed on numeric values. These are the most common:

+
addition
-
subtraction
*
multiplication
/
division
%
modulus (remainder of a division)
++
increment (add 1)
--
decrement (subtract 1)

                val price  = 130.00
                val rate   = 0.175
                val paid   = 180.00
                val people = 6

                val tax    = price * rate    // 22.75
                val total  = price + tax     // 152.75
                val change = total - paid    // 27.25
                val each   = total / people  // 25.46
                people++                     // 7
                people--                     // 6
                people += 5                  // 11
                people % 5                   // 1

Note that if you perform maths on two Ints, the result will also be an Int. If you need a Double (decimal) result, at least one of the values needs to be a Double.

Comparision Operators

When making comparisions between values, many different operators can be used:

==
is equal to
!=
is not equal to
>
is greater than
<
is less than
>=
is greater than or equal to
<=
is less than or equal to
in
is in a given range
!in
is not in a given range 

                val name = "Jimmy"
                val age = 25

                name == "Jimmy"  // true
                name != "Dave"   // true
                age < 18         // false
                age > 65         // false
                age <= 25        // true
                age >= 50        // false
                age in 15..30    // true
                age !in 15..30   // false

Note that the result of these operations is always a Boolean value: true if the comparision is true, false otherwise.

Logical (Boolean) Operators

Sometimes we need to combine multiple Boolean values or operations:

&&
AND (both must be true)
||
OR (either could be true)
!
NOT (true if false, false if true)

                weekend = (day == "Sat") || (day == "Sun")
                canDrive = (age >= 17) && (hasLicense == true)
                hatesBeans = !(likesBeans == true)

Note that the result of these operations is always a Boolean value based on the combination of other values and the operators.

Also note that it is redundant to compare a Boolean value with true or false, since a Boolean value is already true / false. The second and third examples above could be written more succinctly:


                canDrive = (age >= 17) && hasLicense
                hatesBeans = !likesBeans
