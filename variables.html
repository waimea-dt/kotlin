Storing and Using Data in Variables

Variable Types

Kotlin variables can be used to store a range of different data types. There are many, but here are the most common:

  • Int - for integer (whole) numbers, e.g. 1234
  • Double - for decimal numbers, e.g. 27.5
  • String - for text, e.g. "Jimmy"
  • Char - for single characters, e.g. 'A'
  • Boolean - for true / false values

Unlike Python, for example, Kotlin is strongly-typed which means that a variable, once created, can only store that one specific type of data. This can help prevent certain types of error.

More information and examples of different data types can be found here.

Declaring Variables

'Declaring' a variable is what you do when you first create and name it.

When declaring a variable, if you don't provide a data value, you must specify the data type:


                // Declaring some variables with types
                val name: String
                val age: Int
                val height: Double
                val expert: Boolean

If you do provide an initial data value then the data type is inferred (worked out) from the data:


                // Declaring some variables, inferring types
                val name = "Jimmy"   // name is now a String
                val age = 21         // age is now an Int
                val height = 1.85    // height is now a Double
                val expert = true    // expert is now a Boolean

Read-Only Variables (Values)

If you create a variable using the val keyword, its value can be set once, but then never changed (it is 'immutable').


                val name = "Jimmy"
                val age = 21

                name = "James"  // Error as values can't change
                age = 30        // Error as values can't change

Use val whenever possible, unless you know the value might need to change, then use var (see next section).

Modifiable Variables

If you create a variable using the var keyword, its value can be modified whenever needed (it is 'mutable').


                var name = "Jimmy"
                var age = 21

                name = "James"  // Variable value will change
                age = 22        // Variable value will change

Only use var if you know that the value might change. Otherwise, use val (see previous section).

Naming Variables

.


                // Some good variable names
                val name = "Jimmy"
                val name1 = "Jimmy"

                // Good variable names with multiple words
                val firstName = "Jimmy"  // Camelcase
                val first_name = "Jimmy" // Snakecase

                // Some bad variable names
                val n = "Jimmy"          // Not descriptive
                val Name = "Jimmy"       // No capital at start
                val 1name = "Jimmy"      // Only letters at start
                val firstname = "Jimmy"  // Hard to read
